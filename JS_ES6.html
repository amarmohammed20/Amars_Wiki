Let and Const Variables

Javascript has three types of variables Const, Let and var

  • var is global scope and let is block scope. Always use let instead of var
  • The let and const variables were introduced in ES6. VAR is the oldest type of variable.
  • To reassign a value in javascript you can use +=
  • const variables - The value of the variable should remain constant. Example const name = "Amar";. Once assigned a value you cant later down the code assign another value to that variable.
    What you can do though is add values in a object that is assigned to a const variable. Same rule applys for arrays against a const variable. You can update the array.
  • Let and Const resolves the issue we had with var variables were you could redeclare the variable later. You can still ant redeclare the variable later but can reassign the variable using Let and add to the variable Const
  • Let and Const are block scope variables. This means that if we declare a variable inside the brackets {} it can not be used outside the block. ES5 would allow you to use that variable outside of the block scope.
  • Let and Const are block scope variables
  • let variables lets you update/reassign the value of the variable.
  • For a const variable you have to assign a value
  • A variable name can only include letters, numbers, _ and $
  • A variable name cannot start with a number
  • For multi word variables you use camel case which is the second,third and so on word starts with a captial

Prototypes Explained

All JavaScript objects inherit properties and methods from a prototype.

  • All JavaScript objects inherit properties and methods from a prototype:
    • Date objects inherit from Date.prototype
    • Array objects inherit from Array.prototype
    • Person objects inherit from Person.prototype
    • The Object.prototype is on the top of the prototype inheritance chain: Date objects, Array objects, and Person objects inherit from Object.prototype.
  • The proto type against a object will be displays __proto__:object. The __proto__ property of Object.prototype is an accessor property (a getter function and a setter function) that exposes the internal [[Prototype]] (either an object or null) of the object through which it is accessed.
  • Properties that can change within the object should not be in the prototype and things that dont change like a calcualtion for the age of someone will need to be place into the prototype
  • To place something into the prototype of a object you do object.prototype.property = ;. This could be a function of a string or what ever you like.
  • When you console log the object the property that is placed into the prototype wont be seen but will be under __proto__
  • You can manipulate the properties in prototype. This will only work if you reference a property from the object and this property is later changed or reassigned.
  • WHY USE A PROTOTYPE?
