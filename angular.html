What is Angular

Angular is a full featured JavaScript Framework created and maintained by Google and is used for building front end applications or the front-end element of a full stack application

  • AngularJS lets you extend HTML with HTML attributes called directives. There are three kinds of directives in Angular:
    • Components—directives with a template.
    • Structural directives—change the DOM layout by adding and removing DOM elements.
    • Attribute directives—change the appearance or behavior of an element, component, or another directive.
  • Angular is popular in large enterprises
  • Angular is the next generation framework where each tool was designed to work with every other tool in an interconnected way
  • REACT is currently more popular JS frameworks
  • AngularJS is not Angular and AngularJS should be avoided
  • Currently on version 7 of Angular
  • Some reasons to use Angular
  • Angular is a all in one solution
  • Can be used to build web and mobile applications both IOS and android

Setting Up an Angular application

You are about to embark on an extraordinary journey. Writing JavaScript is a lot of fun, but writing an Angular 2 application with TypeScript is pure joy. In this course we'll cover the basics of building an Angular 2 application.

  • Sound cloud is a angular app which taps into the soundcloup api. The Api links to a database were the information is stored
  • Angular is modular giving you control over your code. When passed to the browser it will be in its simpliest form
  • Angular is built on a static lanuage called TypeScript. Static lanuage allows you to spend less time reading documentation and more time building your application

The Parts and Pieces of an Angular Application

  • A Angular application is built from a collection of modules. A app needs to have one root module which is were the application starts
  • Angular consists of Services, Components and 3rd Party Modules
  • Components are made up of the following: templates, Styles and Business Logic
  • Ng is the namespace Angular adopted so developers can easily identify parts of the Angular platform. (example: NgModule, NgForms, ngOnInit)
  • Webpack is used for packing Angular applications. Webpack is a module loader.
