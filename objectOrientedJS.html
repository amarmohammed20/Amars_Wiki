Prototypes Explained

All JavaScript objects inherit properties and methods from a prototype.

  • All JavaScript objects inherit properties and methods from a prototype:
    • Date objects inherit from Date.prototype
    • Array objects inherit from Array.prototype
    • Person objects inherit from Person.prototype
    • The Object.prototype is on the top of the prototype inheritance chain: Date objects, Array objects, and Person objects inherit from Object.prototype.
  • The proto type against a object will be displays __proto__:object. The __proto__ property of Object.prototype is an accessor property (a getter function and a setter function) that exposes the internal [[Prototype]] (either an object or null) of the object through which it is accessed.
  • Properties that can change within the object should not be in the prototype and things that dont change like a calcualtion for the age of someone will need to be place into the prototype
  • To place something into the prototype of a object you do object.prototype.property = ;. This could be a function of a string or what ever you like.
  • When you console log the object the property that is placed into the prototype wont be seen but will be under __proto__
  • You can manipulate the properties in prototype. This will only work if you reference a property from the object and this property is later changed or reassigned.
