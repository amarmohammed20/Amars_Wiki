Getting Started

What is REACT?

  • React runs on the front end such as the broweser. Other lanuages like PHP or python run on the server
  • By running on the browser you get some benifits such as a faster running time. This is becuase when a change a occurs only that part of the page reloads and not the whole page.
  • React uses the concept of plugins for more advanced things which makes it very lightweight
  • React is the most popular framework in the industry right now - 2019
  • Why you should use REACT instead of vanilla JavaScript:
  • REACT uses JSX which stands for JavaScript XML.
    JSX allows us to write HTML in React.
    JSX makes it easier to write and add HTML in React.

  • REACT thinks in components as shown in the below example:

  • Components can have a state and the component changes when the state changes. An exmaple could be that a box has a red border when the questions inside is answered correctly.
  • React at its heart is ran by components, below is an example were we have a website that is built of components. We can reuse some components on other pages like tweet like and nav bar
  • To manage states we can also use Redux - Redux is a predictable state container for JavaScript applications. It helps you write applications that behave consistently, run in different environments (client, server, and native), and are easy to test. Simply put, Redux is a state management tool.
  • You can use Create REACT APP to get started to build a react app. This creates a boiler plate app we can build from.
  • THe boiler plater REACT package gives you a index.html. In this file you will get a div in the body called root. All your components should be child componenets of this.

Dev Environment Setup

  1. To install react you will need to have nodejs installed.
  2. We will be using npm to install third party libraries.
  3. Once nodejs is installed open the terminal and use the following npm i -g create-react-app. i is for install and -g is for global and create react app will give us a boiler app for react on the latest specified version. Normally you shouldnt have to specify the version.
    Once completed you will get a confirmation of the number of packages installed.
  4. Open up VS code and install the following two extensions - Simple React Snippets and Prettier - Code formatter. Reload VS code after

First REACT App

Create REACT App

  • Back in the terminal we can create our first REACT app - npx create-react-app test-react-app - test-react-app being the name I gave to the new react app.
  • The above command does everything for you so you dont need to worry about anything:
  • If you want t ocustomize some of the setting you can use npm run eject
  • Navigate to the new folder called test-react-app and than run the following command npm start. This should open up the browser with the localhost and port 3000
  • In VS code we will have a folder called node modules which will hold all the third party libraries as well as REACT itself. We should never have to touch this folder.
  • We have a public folder that holds our public assests such as our browser icon main index html page
  • In the HTML page index.html you will see some standard html boiler code. What you need to look for is in the body, a div that has the id of root. This is were our react application will start from, live and grow.
  • We have a app.test.js file which holds and controls the component we see on the opening page the black banner. The output of that component in app.test.js is what you see in the black banner.
  • You will see in the file app.js we are using es6 classes. You will see a meathod called render which you will see spits out JSX (JavaScriptXML), not HTML or JS.
  • To make JSX work we have to pass it through babel which is a modern JS compiller

Custom Configs

  • To perform any customizations to the standard setup REACT gives you you use the command npm run eject.
  • Ejecting is a oneway street once one it cannot be undone.
  • Before you eject your package.json file would look simple as REACT package would hide the complex things like web pack
  • After you have ejected you will see all packages and thier configuration. We will also get a new folder called config.

Full-stack Architecture

  • Every App has two parts a front end and back end. Our focus is on the front end as REACT is a front end library
  • To get a easy backend to your REACT app you can use Firebase
  • Your REACT app will normally talk to a backend which will hold a database.
  • The best resources you should use for the back end are nodeJS and express.
