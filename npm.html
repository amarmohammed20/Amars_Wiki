The Basics Of NPM - Node Package Manager

NPM is a node package manager. Nodes are reusable code that is created by other people. Before using npm you will need node.js

You will get npm manager downloaded when you download Node.js. You can check if you have node.js installed or npm by typing node -v or npm -v in the command propmt. This will bring back the version of either node.js or npm

A package.json file is located in every root location of a node project. This file holds various metadata relevant to the project. This file is used to give information to npm that allows it to identify the project as well as handle the project's dependencies. It can also contain other metadata such as a project description, the version of the project in a particular distribution, license information, even configuration data - all of which can be vital to both npm and to the end users of the package.

  • When starting out with working with a Node JS project create a package.json file by uisng the command in your terminal (Be in the correct path) - npm init
  • Keep everything default through the setup of the package.json file
  • The main purpose of the package.json file is to store all of your dependancies. So when you move to another environment you move this file and simply run npm install
    Remember not to move the node_modules folder as all will be reinstalled using the package.json file.
  • You will also get a package-lock.json file that will record all the dependacies and the associated versions
