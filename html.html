Getting started with HTML

HTML is understood by all browsers, from browsers on phones, to tablets to desktop computers. That's why every website and web app you use is made using HTML -- it's the language of the web!

  • HTML provides the structure of you website/app, being the foundation of any great website/app
  • A website has three layers, content (What you see on the page), Presentation Layer(How that presentation looks) and behaviour layer(interaction on the page)
  • Css provides the presentation layer
  • Javascript provides behaviour layer, interaction
  • HTML provides the content layer, forms the structure
  • Markup lanuage provides meaning to text in a document
  • Tags are used in markup to provide formatting and structure to your page

Semantic HTML: header, footer and section

HTML's role in web design and development is to provide structure and meaning to content. In this video, you'll begin learning HTML's set of elements that describe sections of content.

  • Semantic HTML - Markup that describes the meaning of content instead of how it looks
  • A example is that a /p tag and a /h tag but display text but have different meanings
  • Semantic is used to example your inofmration to search engines
  • Semantic is also used for assisted users who might be visually impared
  • Below illustrates Semantics, elements that provide structure to the pages, such as header and footer.
  • The head tag is used for holding Meta information, title, links, etc. and is not displayed on the page. The header tag is used within the body of the website and can be used multiple times if required, e.g. to determine the top of an article.
  • The HTML footerelement represents a footer for its nearest sectioning content or sectioning root element. A footer typically contains information about the author of the section, copyright data or links to related documents.
  • The SECTION element is defined as a semantic section of a web page or site that isn't another more specific type. I tend to use this element when I am marking up a distinct section of the page - a section that could wholesale be moved and used on other pages or parts of the site. It is a distinct piece of content, or a "section" of content, if you will.
    In contrast, you use the DIV element for parts of the page that you want to divide up, but for purposes other than semantics.
  • At the start of every section there should be a heading indicating what that section is about
  • The HTML article element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable (e.g., in syndication). Examples include: a forum post, a magazine or newspaper article, or a blog entry.
  • The nav element is used to provide navigation within the structure, this could be a link
  • The nav element is normally used for main links around the page/website. Like the navigation buttons that take you to different pages
  • The aside element are normally buttons on the side of the page, represents a portion of a document whose content is only indirectly related to the document's main content. Asides are frequently presented as sidebars or call-out boxes.
  • The q tag indicates that the enclosed text is a short inline quotation. Most modern browsers implement this by surrounding the text in quotation marks. This element is intended for short quotations that don't require paragraph breaks; for long quotations use the blockquote element.

Grouping Content with main and div elements

  • The HTML main element represents the dominant content of the body of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application.
  • The HTML Content Division element div is the generic container for flow content. It has no effect on the content or layout until styled using CSS.
  • The HTML Citation element cite is used to describe a reference to a cited creative work, and must include the title of that work. The reference may be in an abbreviated form according to context-appropriate conventions related to citation metadata.
  • You should use the main tag once in a page and not include information the can be repeated across other pages
  • The div element is used as a container and has no semantic meaning at all
  • Using a div at the start of the body is called uisng a wrapper and it controls things like the width alignment and so on.
  • blockquote tag is used to block a long quote, cite can be used to were you got the information from. You can wrap the cite in a footer attributing the text to the author

Using multiple header and footer elements

You can use multiple footer and header elements in one page. You nest footers inside other elements to contain footer information for that section of the page. Nested headers represent introductory content for a section of the page.

  • You can have multiple footer elements. The typical example would be one footer tag in the body for the main footer for the page and than you can have a footer in another element which will be a footer for that section.
  • The same comcept for multiple footer tags apply to multiple header tags, not head tags.

Add Meaning to Words with Text Level Elements

Sometimes you'll want to give a piece of text emphasis or importance. HTML provides semantic elements, like strong, em and small, for formatting text. Other elements like span are used to mark up text just for styling purposes.

  • strong element is used to show important parts of text
  • em tags apply emphasis to text and sets the text to italic.
  • The small tag is used for fine print things legal disclaimers and so on
  • The span element is like the div element it has no semantic meaning. It allows you to lace styling to a section of text

HTML Entities and Reserved Characters

Certain characters are reserved for use in HTML code only. If you use reserved characters in your content, the browser will interpret them as HTML code, and the characters will not appear in your content as expected.

  • To display a reserved character you have to use a entity
  • All the HTML reserved charecters and related entaties can be seen here Character entity chart
  • The HTML pre element represents preformatted text which is to be presented exactly as written in the HTML file. The text is typically rendered using a non-proportional ("monospace") font. Whitespace inside this element is displayed as written.
  • To apply more space between words you can use the entity   nbsp
